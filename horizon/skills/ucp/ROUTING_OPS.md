# 路由表操作规范

## 注册与移除

注册与移除是对路由表的操作，skill 目录本身不受影响。

### 注册

当 `.skills/` 下出现新的 skill 目录时，将其注册到路由表。**以下步骤必须按顺序执行**：

**第一步：前置检查**
1. 确认目录包含 `SKILL.md`，且 frontmatter 中有 `name` 和 `description`
2. 若为 `shared` 源，确认 frontmatter 包含 `origin` 字段

**第二步：提取关键词和正则**
1. 从 `SKILL.md` 的 `description` 和适用场景中提取 `trigger_keywords`（至少 5 个，覆盖典型用户表述）
2. 若关键词不足以精确区分意图，补充 `trigger_patterns` 正则
3. 将关键词与路由表中所有已有条目的关键词和正则逐一比对，检查是否存在误匹配：
   - 某个关键词是否也会命中不相关的已有 skill？
   - 某个正则是否与已有 skill 的正则重叠？
   - 若有冲突，调整关键词或正则使其更精确

**第三步：分配路由属性**
1. 分配 `priority`，不与已有 skill 冲突（同优先级时按路由表顺序先后）
2. 标注 `source`

**第四步：写入路由表**
1. 在「当前路由表」中添加一行

**第五步：验证**
1. 确认 `.skills/<skill_name>/SKILL.md` 文件存在且可读（skill_name 必须与实际目录名完全一致，包括版本后缀）
2. 用至少 2 条**应该命中**的典型用户消息测试——每条消息应正确命中新 skill
3. 用至少 1 条**不应命中**的典型用户消息测试——应验证没有误命中新 skill
4. 验证不影响已有 skill 的优先级排序——已有 skill 在各自典型消息下仍能正确命中

**第六步：一致性校验**

执行以下检查，全部通过才视为注册完成：

1. **目录可达性**：`.skills/<skill_name>/SKILL.md` 存在且可读
2. **frontmatter 必填字段**：`name`、`description` 存在；若 `source=shared`，`origin` 也必须存在
3. **路由表双向一致**：
   - 路由表中每个 `skill_name` 都能在 `.skills/` 下找到同名目录
   - `.skills/` 下每个含 `SKILL.md` 的子目录都在路由表中有对应条目
4. **关键词/正则无冲突**：新条目的 `trigger_keywords` 和 `trigger_patterns` 不会命中不相关 skill 的典型消息（第二步已做，此处复核）

校验不通过时：撤回路由表新增行，视为注册失败，需修正后重新走完整流程

### 移除

当 skill 不再需要参与路由时，将其从路由表中彻底删除。**以下步骤必须按顺序执行**：

**第一步：确认无其他引用**
1. 在所有 `.skills/*/SKILL.md` 中搜索被移除 skill 的 `skill_name`，确认仅有其自身引用

**第二步：从路由表删除**
1. 在「当前路由表」中删除该 skill 对应的整行

**第三步：一致性校验**

执行以下检查，全部通过才视为移除完成：

1. **路由表双向一致**：
   - 路由表中每个 `skill_name` 都能在 `.skills/` 下找到同名目录
   - `.skills/` 下存在但不在路由表中的目录属正常，不报错
2. **无悬空引用**：路由表中无任何条目的文档或描述引用了被移除的 skill_name

校验不通过时：撤回路由表删除（恢复该行），视为移除失败，需修正后重新走完整流程

校验通过后，skill 目录保留不动（如需删除目录，需用户二次确认）

## 多源规范

| source | 说明 | 目录方式 | frontmatter 额外字段 |
|--------|------|---------|---------------------|
| `local` | 本项目专用，与项目代码强耦合 | 直接放在 `.skills/<name>/` | 无 |
| `shared` | 跨项目共享的通用 skill | 符号链接或拷贝至 `.skills/<name>/` | `origin` |

```bash
# shared 引入示例
ln -s /path/to/shared/skills/<name> .skills/<name>
```
