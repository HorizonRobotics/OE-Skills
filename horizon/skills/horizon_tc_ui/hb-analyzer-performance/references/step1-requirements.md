# 第 1 步：理解需求

在运行分析之前，需要收集以下信息：

## 必需信息

### 1. 模型文件
询问用户：
- 模型文件路径是什么？
- 模型格式：.onnx / .bc / .hbm

**如何检查**：
```bash
ls *.onnx *.bc *.hbm 2>/dev/null
```

### 2. 目标架构（march）

**仅对 .onnx 和 .bc 模型必需**，.hbm 模型会忽略此参数。

常见架构：
- `nash-b` - J6B 平台
- `nash-e` - J6E 平台
- `nash-m` - J6M 平台
- `nash-p` - J6P 平台
- `nash-h` - J6H 平台
- 其他：参考项目文档

询问用户：
- "目标架构是什么？(nash-b/nash-e/nash-m/nash-p/nash-h)"

### 3. 分析类型

有三种分析模式：

**模式 A：本地模型分析**
- 输入：模型文件 + march
- 输出：仿真性能数据

**模式 B：板端实测**
- 输入：模型文件 + march + 板子IP
- 输出：真实硬件性能数据
- 需要：SSH 访问权限

**模式 C：已有 perf JSON**
- 输入：perf_result.json 文件
- 输出：性能分析报告
- 适用：已经运行过板端测试

询问用户选择哪种模式。

## 可选信息

### 板端测试参数（模式 B）
- IP 地址：板子的网络地址
- 用户名：默认 `root`
- 密码：默认为空
- SSH 端口：默认 `22`

## 示例对话

```
用户："分析这个模型的性能"

你的回应：
1. 先检查当前目录的模型文件
2. 如果找到模型：
   "我看到有 resnet18.onnx，是要分析这个模型吗？"
   "目标架构是什么？(nash-b/nash-e/nash-m/nash-p/nash-h)"
   "需要板端实测吗？如果需要，请提供板子 IP"
3. 如果没找到：
   "请提供模型文件路径"
```

收集完信息后，进入第 2 步。
